Using The Program

Accessing The Windows File System

When ADF Opus starts you will see a Windows lister already open. This lister provides access to the files on your PC. You can choose whether this lister should open with a list of drives or in a particular directory in the "Options" dialogue. To open a new Windows lister, select View|New Windows Lister .

Opening A Disk Image

To work with an ADF, ADZ or DMS file you must first open it. You can open one by doing any of the following:

· select Open... from the File menu

· click the button on the toolbar

· press Ctrl+O

Then select the file from the dialogue box that appears. Note that it is possible to open more than one disk image at a time - each will open in its own window.

Navigating

Navigation is simple - just like Windows Explorer. Double-click a directory to change into it and view its contents. To go back to the parent directory, click the button on the toolbar.

To change to a different drive in the Windows pane, just click the button when you're already at the root directory. Icons for all drives will appear - just double-click one to view its contents.

Copying Files

Copying is dead easy - just drag and drop the file from one pane to another and it will be copied.

Note: drag and drop is not restricted to a single window - if you have two disk images open at once you can copy files from one to another. To move files within a disk image you can open two copies of a single disk image and drag files from one to the other. You can even copy from one Windows directory to another (but you might be better off using Windows Explorer for managing your Windows files). Go ahead and try it - knock yourself out!

Creating A New Directory

To create a new directory, either select Action|New Directory or click the button on the toolbar.

Renaming Files And Directories

You can rename a file or directory by selecting it and then selecting Action|Rename or clicking the button on the toolbar.

Deleting Files And Directories

You can delete a file or directory by selecting it and then selecting Action|Delete or clicking the button on the toolbar.

Undeleting Files And Directories

Undeleting files or directories is a two-step process. Firstly, with the parent ADF's window active, select View|Undeletable Files or click the button on the toolbar. Opus will then display all files available for undeletion or advise otherwise. Undeletable files are shown ghosted. Undelete an entry by selecting it and then selecting Action|Undelete or clicking the button on the toolbar.

Note: ADFLib's undeletion routines are imperfect. All undeletable files are notified at the root level as well as in the directory from which they were deleted. You need to be in the correct directory for undeletion to succeed. You may have to find the correct one by trial and error. Displaying all files at the root level actually acts as a summary of available recoverable files, so it's not all bad. Some files that should be recoverable may not be displayed. Some files that are displayed may not undelete successfully. Unfortunately, I can't fix this. I still think that some level of recovery is useful, rather than having none. At least you have some chance of being able to recover lost data.

Creating A New Blank ADF

Choose File|New... or click the button to open the Create New Volume dialogue. You can set the path, label and other options from this dialogue. You can also choose whether to use FFS, Directory Caching and International Mode by selecting the appropriate checkboxes under Flags . You can make an ADF bootable by selecting Install from the Tools menu or clicking the button.

Caution: ADFLib can not detect whether an ADF is already bootable or the existing bootblock is valid. To avoid losing anything of value, work on a copy of the ADF and check it within an emulator before deleting the original.

Creating A New Blank Hardfile

Choose File|New... or click the button to open the Create New Volume dialogue. You can set the path, label and other options from this dialogue. You can choose the size of the hardfile by selecting Preset Size and selecting a size from 1 to 512MB from the lister, or by selecting Custom Size and entering the required size manually. You can also opt to use FFS, Directory Caching or International Mode, as for ADFs.

Note: For maximim compatibility, especially with older versions of Workbench, leave FFS and International Mode turned off.

Getting Information About A Disk Image

Choose Information... from the File menu or click the button to view a summary of the current ADF such as how much free space there is, what file system it uses, whether it has been compressed and so on. See the Volume Information Dialogue section for details.

Getting Information About A File Or Directory

Choose Properties... from the Action or context menus or click the button to display the attributes or property flags for a file or directory. If the selected item is an Amiga file or directory, its file comment will also be displayed, if one exists. Comments are limited to 79 characters in length by ADFLib. See the Properties Dialogue section for details.

Closing A Disk Image

This is pretty obvious but when you're finished with a disk image you need to close it before you use it in another program like UAE. To close an image just select its window and click the button on the toolbar (or select File|Close ). If you don't close an ADZ you have made changes to, the changes will be lost at program shutdown. All changes to DMS files will be lost since ADF Opus can't compress to DMS format (see Introduction). If you want to modify a DMS file, convert it to ADZ and modify that.

Running Or Opening Files

You can run or open files listed in the file listers by double-clicking the file. Files appearing in an Amiga lister will open as long as the file type is registered with Windows and a suitable application exists which can handle that file type. For example, if you double-click on a text file called myfile.txt and the .txt file type is registered to open in your text editor, myfile.txt will open in the text editor. See Registering File Types With Windows

Viewing Files Within ADF Opus

Select the file and then choose Tools|Text Viewer or click the button to display the text viewer. The viewer can also be activated from the context menu. See the Text Viewer Dialogue section for details.

Temporary Files

When ADF Opus opens a compressed disk image, it creates a temporary, intermediate ADF in specially created directory called opustemp . If the image's child window is specifically closed, Opus recompresses an ADZ, or simply closes a DMS, and deletes the related ADF from the opustemp directory. If ADF Opus is shut down with child windows still open, any changes to an ADZ will be lost and the temporary ADF will not be deleted normally. To avoid leaving ADFs in the temp directory, ADF Opus removes all ADF files from the opustemp directory on shutdown.
